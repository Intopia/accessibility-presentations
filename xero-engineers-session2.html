Assistive technologies and forms

It is important to understand how screen readers operate, especially when interacting with form elements.

The NVDA and JAWS screen readers have two different modes:

  • Browse mode
  • Forms mode (JAWS)/Focus mode (NVDA)

‘Browse mode’

‘Browse mode’ is used when navigating around and reading documents or web pages.

Some keystrokes can be used as navigation shortcuts in ‘Browse mode’. For example, the letter H will take users to the next heading.

‘Forms/Focus mode’

‘Forms/Focus mode’ is used when interacting with form fields that require user input.

When tabbing in ‘browse mode’ and reaching an element requiring keyboard input, JAWS and NVDA automatically switch to ‘forms/focus mode’.

How do screen reader users navigate forms?

There are many ways these users may navigate through forms.

Method 1:

Reading line-by-line through the form in ‘Browse mode’ to understand the form’s purpose.

Method 2

Navigating via specific keystrokes (in JAWS and NVDA) to jump directly to form controls.

Key Task
F Next form field
B Next button
R Next radio button
X Next checkbox

After using the F key to navigate to a form field, users can use ENTER to switch into ‘Forms/Focus mode’ if they need to type data into the field.

Method 3

Navigating via the TAB key, allowing the user to move to the next element that can receive focus - links, buttons, form fields.

‘Forms/Focus mode’ and form content

There is something to remember when a user navigates through a form while in ‘Forms/Focus mode’ (i.e. using F or TAB keys).

Any content within the form that cannot receive focus or is not programmatically associated with a form field will not be announced.

For this reason, any information relevant to a form field must be programmatically associated with that field.

  • Label text
  • Radio group and checkbox group headings
  • Form field instructions
  • Form field error messages
  • Required field notifications
