What are screen readers?

Screen readers are assistive technologies that help specific groups of people to access digital content.

Screen readers are designed to convert web page content into either of the following:

  • Synthesised speech so that users can listen to the content.
  • Braille so that users can read the content as braille output.

Since the majority of screen reader users don’t use a mouse, almost all screen readers use a wide variety of keyboard commands to carry out different tasks.

There is a common misconception that screen readers ‘read what’s on the screen’.

In reality, screen readers interact with a version of the page source code, referred to as the Accessibility Tree.

Another common misconception is that screen readers are only used by people who are fully blind.

In fact, lots of different types of people use screen readers including:

  • People with no vision
  • People with reduced vision
  • People with reading issues such as dyslexia

Types of screen readers?

There are a wide range of screen readers across different operating systems, including the following:

Operating System Screen Reader
Windows Microsoft Narrator
Windows JAWS - Job Access With Speech
Windows NVDA - NonVisual Desktop Access
Apple OSX/iOS Voice Over
Android TalkBack
Web-based ChromeVox - Chrome Screen Reader

There are also applications such as ZoomText Fusion, which combines ZoomText and JAWS into one application.

How screen reader users navigate

Screen reader users navigate around web pages in different ways depending on their need. This includes:

1. Reading the page from top to bottom

This is done by allowing the screen reader to announce all content on the page.

2. Skimming through content

This is done by skimming over content, jumping onto the next line before the current line has finished being announced.

3. Keyboard shortcuts

Using keyboard shortcuts to navigate around the page, via Headings, Links, Forms, Buttons, Tables, Landmarks, Lists and more.

4. Search

Search for specific information within the page using either:

  • Browser-based search functionality.
  • Specific search functionality within the screen reader.

5. Focusable elements

Using TAB keystrokes to navigate around the website or web app via focusable elements. This is most commonly used in forms.

Common keyboard-only pain points:

1. Lack of keyboard-only access

Users cannot access key site functionality without full keyboard access.

2. Inability to bypass blocks

Users cannot quickly navigate to locations in web pages, and have to listen to every focusable element announced..

Common Screen reader pain points:

1. Lack of semantic markup

Cannot navigate via headings, landmarks.

2. Lack of intuitive accessible name

Cannot understand the purpose of the component.

3. Lack of accessible form markup

Cannot complete forms without effort.

4. Not informated of changes

Some part of the content or UI has dynamically changed, but the assistive technology has not been informed.
