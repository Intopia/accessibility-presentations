Accessible Images

Why are images important?

There is a common misconception that images make digital content less accessible.

But in fact, images can be incredibly helpful.

Especially when they are used with purpose and described well.

Images can improve understanding, support navigation, and help communicate key messages.

They are especially useful for people with language, processing, or memory impairments.

Visuals support recognition over recall, which can reduce cognitive load.

Four types of images

There are four common types of images used in digital content, and each has different accessibility requirements:

  • Type 1: Decorative images
  • Type 2: Informative images
  • Type 3: Functional images
  • Type 4: Complex images

Type 1: Decorative images

Decorative images don’t convey important content.

If they were removed, the meaning of the page would stay the same.

The following image contains text that also appears on screen nearby. This is an example of a decorative image.

From an accessibility perspective, decorative images should be hidden from assistive technologies.

This can be achieved using a null (or empty) alt attribute like this:

<img src="a.png" alt="">

Avoid leaving out the alt attribute entirely, as some screen readers may announce the file name instead.

In the following code example, there is no alt attribute provided at all:

<img src="a.png">

Type 2: Informative images

These are the most common type of image.

Informative images convey important information that supports the content.

They must include alternative text that expresses the meaning or purpose of the image, not just what it looks like.

And here is a code example:

<img src="a.jpg"
  alt="

    environmental sustainability 
    standing in front of the
    Sydney Opera House"
>

Type 3: Functional images

Functional images are used inside interactive elements, such as links or buttons.

The alternative text must describe the action or destination — not the visual appearance.

Here is an example, where an icon is positioned inside a button, in the top right corner of a modal dialog:

In the following bad code example, the visual appearance of the image is described:

<button>
  <img src="a.png" alt="Cross icon">
</button>

Describing the image itself does not help the user understand what it will do.

In the following good code example, the action is described:

<button>
  <img src="a.png" alt="Dismiss">
</button>

Logos used as links should also have functional alt text — such as “Company name home.”

Here is an example of the Salesforce logo, which may be positioned inside a link:

And here is the code example:

<a href="#">
  <img src="logo.png" alt="Salesforce home">
</a>

Type 4: Complex images

These include charts and graphs, maps, diagrams, and infographics.

They need two types of description:

First, they need a short alt text using the alt attribute.

This should identify the image and the location of the longer description.

And second, they need a longer text description.

This can be provided either on the same page or via a link nearby.

Here is an example of a complex diagram:

And here is the code example:

<img src="a.png"
  alt="Venn diagram depicting content
  category relationships. 
  Detailed description below."
>

Avoid placing long descriptions inside the alt attribute.

Screen readers will read the whole thing aloud, which can be overwhelming and difficult to remember or interpret.

The longer description can appear in the following locations:

  • In surrounding text on the same page
  • In a visible caption
  • Or on a separate page linked near the image

Alt text vs captions

Alt text provides an alternative for people who cannot see the image.

For example, screen reader users or people with low bandwidth.

Captions provide visible context for everyone.

Avoid duplicating content in both.

If the caption and alt text are the same, screen reader users will hear the same information twice.

And here is difference between the alt text and caption:

  • Caption: “Ticket revenue had dried up amid coronavirus travel bans.”

Images that contain text

Images that contain text need to be treated as informative images.

Images that include readable text must include that same text in the alt attribute.

Otherwise, screen reader users will miss important aspects of the message.

Alt text is contextual

Alt text depends on the context of the image — why it’s there and what it’s meant to convey.

Here is a a photo of Harvard Stadium with a person running up some steps.

On a general page, the alt text may be:

“A nearly empty Harvard Stadium.”

On a track try-outs page, the alt text may be:

“Harvard Stadium with a man running up stone steps.”

On a renovations page, the alt text may be:

“Harvard Stadium with cracked concrete pillars.”

Tips for writing effective alt text

Here are some tips to help write effective alt text.

1. Think about the image’s purpose and context.

2. Be concise, but clear and meaningful.

3. Use plain, direct language.

4. Put the most important information first.

5. Screen readers already announce that something is an image.

So, avoid phrases like “image of” or “picture of” unless it’s a specific content type like a graph or comic where this affects meaning.

How long should alt text be?

There’s no official character limit for alt text in HTML.

The old 125-character myth comes from outdated screen reader behavior.

Alt text should be as long as it needs to be to convey meaning — but no longer.

If lengthy detail is needed, use a longer description elsewhere.

Some platforms or content management systems may impose character limits — so check your tools.

Conclusion

When in doubt, ask yourself: “What would someone miss if they couldn’t see this image?”

That’s the story your alt text needs to tell.
